# Hotspot Hub Architecture

## Server and runtime

- `src/server.ts` is the main server entry.
- The server handles API requests under `/api/*` and forwards all other requests to the TanStack Start SSR handler.
- `vite.config.ts` configures the TanStack Start server entry to use `src/server.ts`.
- `wrangler.jsonc` is configured for Cloudflare deployment with `src/server.ts` as the worker entry.

## API layer

- `src/lib/mongo.ts` provides:
  - `getDb()` to connect to MongoDB
  - `ensureDbSeeded()` to seed mock collections
  - `handleApiRequest()` to route API paths
- Supported collections:
  - `nodes`
  - `users`
  - `vouchers`
  - `advertisements`
  - `transactions`
  - `activeSessions`
  - `notifications`
  - `revenue7d`
  - `sessionsHourly`
- The API includes one combined endpoint: `/api/dashboard`

## Data model

- Mock data lives in `src/lib/mock-data.ts`.
- Active advertisement schema is defined as `Advertisement`.
- The `advertisements` collection uses a schema with fields such as:
  - `ad_id`
  - `ad_title`
  - `campaign`
  - `campaign_start`
  - `campaign_end`
  - `target_nodes`
  - `ad_url`
  - `target_url`
  - `ad_type`
  - `ad_duration`
  - `custom_button_text`
  - `active`
  - `cpv_rate`
  - `budget`
  - `total_limit`
  - `daily_limit`
  - `daily_count`
  - `impressions`
  - `bill_id`
  - `bill_date`
  - `bill_type`
  - `bill_paid`
  - `cpn`

## Client data access

- `src/lib/api.ts` defines fetch helpers for each backend endpoint.
- `src/routes/_app/ads.tsx` renders advertisement summaries and campaign details.
- React Query is used for data fetching on the ads page.

## Route structure

- App routes are defined under `src/routes/`.
- The ads admin page is `src/routes/_app/ads.tsx`.
- The dashboard and other pages follow the route structure generated by `routeTree.gen.ts`.
